Program Overview
I used the default solution provided in the project guidelines document and used row/column swapping to mess up the solution to create a new random solution.
Data Structures
The 2D array: I used a 2D array to store the game data board.
 The Array and/or ArrayList: I used an ArrayList (3 times) in order to have a way to quickly store and remove possible valid digits and ensure that everything in a certain region/row/column was in the right place.


Algorithm Design
The main algorithm is decided by the method “backToTheFuture”. I choose two random numbers from 0-8 (inclusive) using Math.random(). Those two numbers point to either two rows, two columns, or one of the 9 minizones in the sudoku. I then choose between swapping rows or columns in the sudoku using Math.random() again, I run the swap and I check if the sudoku is valid using “checkValids()”. The checkValids() checks to see if the new solution is good by checking one of the swapped columns (or just a random column if columns weren’t swapped), one of the swapped rows (or just a row if rows weren’t swapped), and the minizone of the location of (a, b). Honestly I didn’t think checking valid solutions like this would work but it seems like it does so I’m happy. If we changed the solution 100 times then we return “true” and stop the algorithm before displaying our new solution. If we haven’t reached 100 then we return the backToTheFuture() function again until it hits 100.


Methods
main() just runs the program
backToTheFuture() is my backtracking algorithm that decides what to do
swapCols() swaps columns
swapRows() swaps rows
miniMidPoint() finds the middle of the minizone that the point is in
checkValids() calls 3 other functions to check if the board is valid (honestly i think its like a 99.9999% chance that the board is valid, there’s probably  a few very rare moments where it’d break but this works fine)
validsInCol() checks to see if the numbers in a column are correct (1 of each number from 1-9 inclusive)
validsInRow() checks to see if the numbers in a row are correct (1 of each number from 1-9 inclusive)
validsInMiniZone() checks to see if the numbers in a minizone are correct (1 of each number from 1-9)
kill() is used for debugging to shut down the program immediately
display() displays the grid
